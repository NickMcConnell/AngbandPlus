# This file defines hard-coded wilderness maps. $WORLD of 0 will be
# used for randomly generated world maps (TODO). You can allow the
# player to move from world to world by offering a suitable town 
# service to increment (or adjust) p_ptr->world_id. There is no longer
# a "wilderness travel mode" so you probably want to keep your maps
# small and let the user "travel" to new worlds as they gain in power
# or accomplish specific goals.
#
# We are re-using the normal room/vault/quest/town parsing engine.
# Letters in D_WORLD will create a dun_grid_t as follows:
#   feat = Feature of the terrain on this grid
#   special = The number you enter; typically the level of difficulty
#   info & CAVE_ROAD => this grid has a road (direction inferred)
#   info & CAVE_TOWN => special gives the town_id
#   info & CAVE_DUNGEON => special gives the dun_type_id
# A grid can no longer give both a town and a dungeon entrance.
# The player will start on a MARK'ed TOWN grid, so only give one.
# Note: Only 1 town allowed per 3x3 grid with the current building
# implementation (they are global and re-parsed all the time).
#
# This file is only parsed on dun_mgr_birth() at the moment. The
# resulting D_WORLD dun_t is then saved in the savefile. The D_WORLD
# will be mapped as the user travels about, CAVE_MARK'ing grids in
# D_WORLD. To travel to a new world, I need to remove the existing
# D_WORLD map and generate a new one. (XXX this needs to be done!)

# World = 1 is the old hengband world. Too big for real use, it is
# here now for testing purposes XXX D s/b DESERT; & s/b VOLCANO; etc
