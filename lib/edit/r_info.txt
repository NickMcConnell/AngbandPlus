# Do not modify this file unless you know exactly what you are doing,
# unless you wish to risk possible system crashes and broken savefiles.

# === Understanding r_info.txt ===

# XXX syntax under construction ... the following is probably out of date XXX

# N: unique internal name (id) : display name
# G: symbol : color
# I: speed : hit points : vision : armor class : alertness : weight (lbs)
# W: depth[ to max_depth]:rarity:exp[:EVOLVE(<id>, <exp>)]
# W: where(depth[ to max_depth]):rarity:exp[:EVOLVE(<id>, <exp>)]
# B: {mon_blow}
# S: spell frequency |
# S: spell type | spell type | etc
# F: flag | flag | etc
# D: Description

# Possessor Extra Information (Optional)
#  The token after P: is a literal an must match case
#  The Copy option, if used, should be specified first.
#   Subsequent options increment this 'baseline', except for Life
#   which is applied multiplicatively.
#  As always, case matters for string lookups (RaceName, ClassName, BodyName).
#  Body types are defined in b_info.txt. If you don't specify one, you get the
#   standard humanoid body type as a default.
# P:Copy:<id> | P:Copy:<RaceName>:<ClassName>
# P:Stats:<Str>:<Int>:<Wis>:<Dex>:<Con>:<Chr>
# P:Skills:<Dis>:<Dev>:<Sav>:<Stl>:<Srh>:<Fos>:<Thn>:<Thb>
# P:ExtraSkills:<Dis>:<Dev>:<Sav>:<Stl>:<Srh>:<Fos>:<Thn>:<Thb>
# P:Life:<Life>
# P:Body:<BodyName>
# P:SpellStat:<Str|Int|Wis|Dex|Con|Chr>
# P:Speed:+12

# 'N' indicates the beginning of an entry. NOTE: The id is no longer
# a 'serial number' but is instead a short string that should be easily
# remembered. This allows reordering of this file, partitioning into
# themed sub-files as well as more readable code and pref files:
# eg '728' vs 'D.storm'. We no longer waste memory on "holes" in the
# id sequence range. Changing the id will break savefiles and might
# also break lookups in EVOLVE directives, quest files and code.

# 'G' is for graphics - symbol and color. There are 16 colors, as
# follows:

# D - Black    w - White     s - Gray     o - Orange
# r - Red     g - Green     b - Blue     u - Brown
# d - Dark Gray  W - Light Gray   v - Violet    y - Yellow
# R - Light Red  G - Light Green  B - Light Blue  U - Light Brown

# 'I' is for information - speed, health, vision in tens of feet,
# armor class, and alertness. 110 is normal speed. Alertness ranges
# from 0 (ever vigilant for intruders) to 255 (prefers to ignore
# intruders).

# 'W' is for more information - level, rarity, and experience for
# killing. The third slot is unused.

# 'B' is for blows
# You can have as many blows as you like and each blow can have
# multiple effects. The first effect is usually plain old damage
# while subsequent effects do things like confuse and stun the
# player. Browse some of the deeper monsters for ideas and to
# learn the syntax.
# Each blow can have multiple attacks per round. For example,
# HIT(x3) will give the monster 3 attacks per round with this
# particular blow. This simplifies lore for identical attacks
# and also avoids duplication here.

# 'S' is for spells. The first S: line should be S:1_IN_X with X the
# number of monster turns, on average, before the monster will cast
# one of its spells. X must not be zero.
# Alt: Use S:FREQ_NN to specify the spell frequency directly.
# Note: You can now give explicit damage amounts for spells.

# 'F' is for flags. These are fairly self-explanatory. As many F:
# lines may be used as are needed to specify all the flags and flags
# are separated by the '|' symbol.

# 'D' is for description. As many D: lines may be used as are needed
# to describe the monster. The parser will insert spaces as needed
# when joining lines.

# 'P' is for posessor information. Generally this should lead with a
# "copy directive" as in P:Copy:Human:Warrior to establish a baseline
# for stats and skills. This also determines the spell stat, if any.
